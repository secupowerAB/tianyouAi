## 本地安装使用

### 方式一：直接使用（最简单）

**步骤1：下载文件**
- 确保你有 `essay-generator.html` 文件
- 保存到任意文件夹（建议桌面或文档）

**步骤2：打开文件**
- 双击 `essay-generator.html`
- 或者：
  - 右键 → 打开方式 → 选择浏览器（Chrome/Edge/Firefox）

**步骤3：开始使用**
- 按照《使用说明.md》配置API密钥
- 即可开始生成作文

**优点：**
- ✅ 无需安装
- ✅ 即开即用
- ✅ 适合个人使用

**缺点：**
- ❌ 无法分享链接
- ❌ 每次需要打开文件
- ❌ 无法展示给评委

---

### 方式二：本地服务器（适合测试）

如果想测试更完整的环境，可以使用本地服务器。

**Python方式（推荐）：**

```bash
# 打开命令行/终端，进入文件所在目录
cd /path/to/your/folder

# Python 3
python -m http.server 8000

# 或 Python 2
python -m SimpleHTTPServer 8000
```

然后访问：http://localhost:8000/essay-generator.html

**Node.js方式：**

```bash
# 安装http-server
npm install -g http-server

# 运行服务器
http-server -p 8000
```

然后访问：http://localhost:8000/essay-generator.html

## GitHub仓库创建

### 为什么使用GitHub？

**优势：**
- ✅ 免费托管代码
- ✅ 版本控制
- ✅ 可以生成在线访问链接
- ✅ 展示技术能力
- ✅ 符合参赛要求

### 创建步骤

**步骤1：注册GitHub账号**
1. 访问 https://github.com
2. 点击"Sign up"注册
3. 填写用户名、邮箱、密码
4. 验证邮箱
5. 完成注册

**步骤2：创建新仓库**
1. 登录GitHub
2. 点击右上角"+"号
3. 选择"New repository"
4. 填写信息：
   - Repository name: `ai-essay-generator` 或 `essay-generator`
   - Description: `AI英语作文生成器 - 2026沈阳市信息素养提升实践活动参赛作品`
   - Public（公开）或 Private（私有）- 建议选Public
   - 勾选"Add a README file"
5. 点击"Create repository"

**步骤3：上传文件**

**方式A：网页上传（简单）**
1. 在仓库页面点击"Add file"
2. 选择"Upload files"
3. 拖拽或选择以下文件：
   - `essay-generator.html`
   - `README.md`
   - `使用说明.md`
   - `安装部署指南.md`（本文件）
   - `演示脚本.md`
4. 在"Commit changes"框中写：`初始提交：添加所有项目文件`
5. 点击"Commit changes"

**方式B：使用Git命令（专业）**

```bash
# 1. 克隆仓库到本地
git clone https://github.com/你的用户名/ai-essay-generator.git

# 2. 进入目录
cd ai-essay-generator

# 3. 复制项目文件到此目录

# 4. 添加所有文件
git add .

# 5. 提交
git commit -m "初始提交：添加所有项目文件"

# 6. 推送到GitHub
git push origin main
```

**步骤4：完善README**
在仓库首页的README.md中添加：
- 项目截图
- 在线演示链接（部署后）
- 使用说明链接
- 作者信息

## GitHub Pages部署

### 什么是GitHub Pages？
GitHub Pages是GitHub提供的免费静态网站托管服务，可以让你的项目有一个在线访问地址。

### 部署步骤

**步骤1：启用GitHub Pages**
1. 进入你的仓库页面
2. 点击"Settings"（设置）
3. 在左侧菜单找到"Pages"
4. 在"Source"（源）下：
   - Branch: 选择 `main`
   - Folder: 选择 `/ (root)`
5. 点击"Save"

**步骤2：等待部署**
- 通常需要1-3分钟
- 页面会显示：`Your site is ready to be published at https://你的用户名.github.io/ai-essay-generator/`
- 刷新几次直到显示：`Your site is live at ...`

**步骤3：访问网站**
- 完整地址：`https://你的用户名.github.io/ai-essay-generator/essay-generator.html`
- 例如：`https://zhangtianyou.github.io/ai-essay-generator/essay-generator.html`

**步骤4：设置首页（可选）**
为了让访问更简洁，可以：

1. 将 `essay-generator.html` 重命名为 `index.html`
2. 重新上传到GitHub
3. 这样访问 `https://你的用户名.github.io/ai-essay-generator/` 就直接打开应用

### 自定义域名（高级，可选）

如果你有自己的域名：

**步骤1：添加CNAME文件**
```
# 创建文件名为 CNAME 的文件（无后缀）
# 内容为你的域名，例如：
essay-gen.example.com
```

**步骤2：配置DNS**
在域名服务商添加CNAME记录：
- 类型：CNAME
- 主机：essay-gen（或@）
- 值：你的用户名.github.io

部署完成后，检查以下项目：

## 常见部署问题

### Q1: GitHub Pages部署后页面空白？
**可能原因：**
- 文件路径错误
- 文件名大小写问题
- 等待时间不够

**解决方案：**
1. 确认文件已上传成功
2. 检查文件名是否正确
3. 等待5-10分钟再访问
4. 清除浏览器缓存

### Q2: 字体显示异常？
**原因：** Google Fonts加载问题

**解决方案：**
- 等待网络加载
- 检查网络连接
- 使用VPN（如果在国内）

### Q3: API调用失败？
**检查：**
1. API密钥是否正确
2. 网络是否可以访问国际网站
3. API额度是否用完

### Q4: 移动端显示问题？
**检查：**
- 使用Chrome移动版测试
- 检查响应式布局
- 调整viewport设置

### Q5: 如何更新已部署的网站？
**步骤：**
1. 修改本地文件
2. 重新上传到GitHub
3. 等待1-2分钟自动部署
4. 刷新网页查看更新
